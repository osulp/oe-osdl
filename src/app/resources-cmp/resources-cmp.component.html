Resources

  • Oregon Geospatial Data Metadata Standard

    The Oregon Geospatial Metadata Standard is based on the Federal Geographic Data Committee Content Standard for Digital Geospatial Metadata.

    You can download a custom ArcGIS Catalog metadata editor to facilitate the creation of metadata that meets the Oregon Geospatial Metadata Standard. The custom ArcCatalog metadata editor was developed by Lane Council of Governments with Framework Data Development Program Funding (2017).

    Download the Oregon GIS Metadata Editor for ArcCatalog.


  • Federal Geographic Data Committee Content Standard for Digital Geospatial Metadata

    For more information about the Federal Geographic Data Committee Content Standard for Digital Geospatial Metadata (FGDC CSDGM), see the FGDC Geospatial Metadata Tools page.

    For information about creating complete FGDC CSDGM metadata in ArcGIS Desktop, see the page "Illustrated guide to complete FGDC metadata."


  • International Organization for Standardization (ISO) Geographic Information – Metadata standard

    For more information about the International Organization for Standardization (ISO) Geographic Information – Metadata standard 19115:2003 see this FGDC webpage.

  • Metadata Best Practices for Map Services

    The same best practices concepts outlined in the Metadata Best Practices links above can be applied when creating map service metadata. Ideally map service metadata will include both metadata describing the map service, and metadata describing the layers within that service.

    When possible, metadata should be applied to ArcGIS server based map services using the steps outlined in ESRI documentation on map service metadata. This metadata describes the map service, so to provide access to metadata for each layer in the service it is recommended that each layer description contain a link to metadata for that layer.
